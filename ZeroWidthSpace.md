# ゼロ幅スペースについて

電話番号の文字数制御の際にゼロ幅スペースの罠にハマった為備忘録

 - 画面では確認できないスペースである
   - 確認はできないけどスペース扱いのため一文字として扱われる
   - HTMLでは、&ZeroWidthSpaceで表示される

ExcelやWord などからのコピペ時に混入するケースがあるよう
# 対処

通常のスペースを消す際に使うtrim関数では消すことができず、<br>
別途preg_replace関数で削除

```
preg_replace('/\x{200B}/u', '', $phoneNumber);←U+200Bがゼロ幅スペースのUnicode
```
